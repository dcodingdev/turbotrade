# Phase 1.1: Next.js Storefront Foundation - Context

**Gathered:** 2026-05-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Establishing the fundamental shopping discovery experience using Next.js App Router and Tailwind CSS v4. This includes the main product discovery page (Storefront) and the Product Detail Pages (PDPs), focusing on SEO and high-performance data fetching.

</domain>

<decisions>
## Implementation Decisions

### Aesthetic & Theme
- **D-01:** Support both Light and Dark mode themes.
- **D-02:** Use the existing theme tokens and CSS variables defined in `apps/web/src/app/globals.css`.

### Product Discovery Layout
- **D-03:** Implement a **Responsive Masonry Grid** for product browsing.
- **D-04:** Ensure the grid adapts seamlessly across mobile, tablet, and desktop breakpoints.

### Data Fetching Strategy
- **D-05:** Use **pure TanStack Query** (Client-side fetching) for all dynamic data (prices, stock, product lists) to ensure consistency across the storefront.

### SEO & Sharing
- **D-06:** Implement **Basic Metadata** only for this phase (title tags, descriptions). Advanced SEO features (Dynamic OG images, JSON-LD) are deferred.

### the agent's Discretion
- **D-07:** Specific component design (e.g., product card interaction details, loading skeletons) is at the developer's discretion, provided they align with the rich aesthetics requirement.

</decisions>
