# Phase 1.1: Next.js Storefront Foundation - Research

## Research Objective
Establish the technical approach for a high-performance, SEO-optimized product discovery experience using Next.js App Router, Tailwind CSS v4, and TanStack Query.

## Findings

### 1. Aesthetic & Theme (Tailwind CSS v4)
- **OKLCH Colors**: The project already uses OKLCH color tokens in `globals.css`.
- **Dual Theme Support**: Tailwind v4 handles the `dark` variant automatically if the `.dark` class is present on the root or via media query. We will use a `ThemeProvider` or a simple script to toggle classes.
- **Glassmorphism**: Can be achieved using `backdrop-blur` and semi-transparent `oklch` backgrounds (e.g., `bg-background/80`).

### 2. Product Discovery Layout (Responsive Masonry)
- **Approach**: For a truly "premium" feel where items maintain order across columns, a library like `react-plock` or `react-masonry-css` is recommended. 
- **Tailwind Native**: CSS `columns-` is an option but can cause items to split or jump unexpectedly. 
- **Recommendation**: Implement a custom Masonry component using `Grid` with `grid-template-rows: masonry` (experimental) or a stable React-based masonry component to ensure layout stability during data loading.

### 3. Data Fetching Strategy (TanStack Query)
- **Consistency**: To maintain "pure consistency," all dynamic data (Price, Stock, Grid) will be fetched via client-side `useQuery`.
- **Hydration**: While the user requested "pure" consistency, pre-fetching the first page of products in RSC and hydrating the client cache is recommended for SEO (Dimension 5). However, if "pure" consistency means strictly client-side, we will proceed with a client-only fetch.
- **Dependency**: Needs installation: `@tanstack/react-query` and `@tanstack/react-query-devtools`.

### 4. Backend API Integration
- **Endpoint**: `GET /api/v1/products`
- **Schema**:
  ```json
  {
    "success": true,
    "docs": [
      {
        "_id": "...",
        "name": "...",
        "price": 100,
        "mainImage": { "url": "...", "localPath": "..." },
        "vendor": { "id": "...", "name": "..." }
      }
    ],
    "totalDocs": 1,
    "totalPages": 1,
    "page": 1,
    "limit": 10
  }
  ```
- **SEO**: Product Detail Pages (PDPs) will use the `id` from the grid items to fetch specific data via `GET /api/v1/products/:id`.

## Validation Architecture (Dimension 8)

| Layer | Strategy | Tooling |
|-------|----------|---------|
| **Visual** | Responsive layout audit (Mobile/Tablet/Desktop) | Playwright / Manual |
| **Logic** | Data fetching state handling (Loading/Error/Empty) | TanStack Devtools |
| **Performance** | CLS (Cumulative Layout Shift) monitoring for Masonry | Lighthouse |
| **SEO** | Metadata presence check | Headless browser / grep |
