# Phase 2.1: Order Management & Timelines - Context

## Overview
This phase transitions the platform from a simple purchase flow to an operational marketplace. It focuses on the fulfillment lifecycle, providing visibility to customers and action-oriented dashboards to vendors.

## Core Requirements
1.  **Backend Enrichment**: Add item-level status tracking and vendor-specific order retrieval.
2.  **Customer Tracking**: Responsive timeline view for order progress.
3.  **Vendor Operations**: Kanban or Timeline view to manage order fulfillment.

## Technical Constraints
- **Multi-Vendor**: Single order, multiple fulfillment paths.
- **Microservices**: Order status changes may need to notify other services (e.g., Stock for confirmation).
