# Phase 2.1: Order Management & Timelines - Specification

## 1. Goal
Implement a robust order tracking and management system that supports multi-vendor fulfillment by tracking status at the individual item level.

## 2. Backend Changes (Order Service)

### 2.1. Model Updates (`order.model.ts`)
- **OrderItem Status**: Add a `status` field to the `items` array in `IOrder`.
  - Enum: `["PENDING", "PROCESSING", "SHIPPED", "DELIVERED", "CANCELLED"]`.
  - Default: `"PENDING"`.
- **Global Order Status**: Maintain `orderStatus` at the top level for overall payment/cancellation state.
  - Updated Enum: `["PENDING", "PAID", "COMPLETED", "CANCELLED"]`.
- **Status Timeline**: (Optional but recommended) Add an `updates` array to track when statuses changed for audit/UI timeline.

### 2.2. New API Endpoints
- `GET /api/v1/orders/vendor`:
  - Returns orders containing items belonging to the authenticated vendor.
  - Items not belonging to the vendor should be filtered out or marked as "other vendor" in the response for privacy.
- `PATCH /api/v1/orders/:orderId/items/:productId/status`:
  - Allows a vendor to transition the status of their specific item.
  - Validates that the vendor owns the item.
  - Triggers global order status update logic (e.g., if all items are `DELIVERED`, the global status becomes `COMPLETED`).

## 3. Frontend Changes (Next.js Web App)

### 3.1. Customer Order Tracking (`(account)/orders`)
- **Order List**: Display all user orders with current global status and total.
- **Tracking Detail**: A dedicated page for a single order:
  - List of items grouped by vendor.
  - Visual timeline for each vendor group (e.g., `Processing -> Shipped -> Delivered`).
  - Progress bar showing the overall journey.

### 3.2. Vendor Order Dashboard (`(vendor)/vendor/orders`)
- **Order Management Table**:
  - Filter by status (Pending vs Processing vs Shipped).
  - Quick actions to transition states.
- **Interactive Timeline/Gantt View**:
  - A visual representation of upcoming and active orders.
  - Ability to drag/click to transition items from `Processing` to `Shipped`.

## 4. Logic & Edge Cases
- **Payment Success**: When `payment-service` signals success, all items in the order move from `PENDING` to `PROCESSING` (or the top level moves to `PAID`).
- **Partial Shipments**: If Vendor A ships but Vendor B is still processing, the customer sees "Partially Shipped" or individual item statuses.
- **Cancellations**: Cancellation logic must handle stock release for specific items if partially cancelled.
