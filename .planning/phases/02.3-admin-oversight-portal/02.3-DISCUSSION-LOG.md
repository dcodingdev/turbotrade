# Phase 2.3 Discussion Log: Admin Oversight Portal

## Design Decisions

### D-01: Tech Stack Consolidation
- **Decision**: Build the Admin Portal inside the existing `apps/web` Next.js application using the `(admin)` route group.
- **Rationale**: Simplifies authentication, component sharing (Shadcn/ui), and deployment. We will prioritize functionality and metrics over a completely separate Vite SPA for now.

### D-02: User Suspension Logic
- **Decision**: Add an `isSuspended` boolean to the `User` model.
- **Action**: Update the login flow to reject users where `isSuspended: true`.

### D-03: Metrics Visualization
- **Decision**: Use simple numeric counters and list views for the MVP. Advanced charting (Recharts) will be considered if time permits.

### D-04: Admin Authorization
- **Decision**: All admin endpoints must be protected by the `authorize([UserRole.ADMIN])` middleware.
