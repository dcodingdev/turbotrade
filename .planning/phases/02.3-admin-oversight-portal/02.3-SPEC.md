# Phase 2.3: Admin Oversight Portal - Specification

## 1. Goal
Provide a centralized administrative dashboard for platform-wide monitoring, user management, and product moderation.

## 2. Backend Changes

### 2.1. Auth Service
- **New Endpoint**: `GET /api/v1/users`
  - Access: Admin only.
  - Returns: Paginated list of all users.
- **New Endpoint**: `PATCH /api/v1/users/:id/suspend`
  - Access: Admin only.
  - Action: Toggles `isSuspended` flag on the user model.

### 2.2. Order Service
- **New Endpoint**: `GET /api/v1/orders/all`
  - Access: Admin only.
  - Returns: Paginated list of all orders across the platform.

### 2.3. Product Service
- **Oversight**: Ensure `updateProduct` and `deleteProduct` continue to support Admin overrides (already partially implemented).
- **Metric Endpoint**: (Optional) `GET /api/v1/products/metrics` to return counts of active vs. draft products.

## 3. Frontend Changes (apps/web - (admin) group)

### 3.1. Admin Dashboard Shell
- Create a shared layout for the admin section with a dedicated sidebar.
- Implement a "Stats at a Glance" view:
  - Total Users (Customers vs. Vendors).
  - Total Products (Published vs. Draft).
  - Total Platform Revenue (Aggregate from all orders).

### 3.2. User Management (`(admin)/users`)
- A searchable table of all platform users.
- Badges for roles (Customer, Vendor, Admin).
- Actions to suspend or unsuspend users.

### 3.3. Product Moderation (`(admin)/products`)
- Global list of all products.
- Ability to view any product detail and "unpublish" (mark as Draft) any item violating platform rules.

### 3.4. Order Monitoring (`(admin)/orders`)
- Global feed of all platform transactions.
- Filter by status (PAID, SHIPPED, etc.).

## 4. Logic & Edge Cases
- **Self-Suspension**: Prevent an admin from suspending themselves.
- **Suspended Login**: Update `auth-service` login logic to block users with `isSuspended: true`.

## 5. Verification Plan
- [ ] Verify Admin-only access to all `/api/v1/...` oversight endpoints.
- [ ] Verify that a suspended user cannot log in.
- [ ] Verify global metrics correctly aggregate data from across the platform.
