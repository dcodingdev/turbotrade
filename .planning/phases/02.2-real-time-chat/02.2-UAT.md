### 1. Cold Start Smoke Test
expected: |
  Kill any running server/service. Start the chat-service. 
  The server should boot without errors, connect to MongoDB, Redis, and log "💬 Chat Service running on port 4006" and "Socket.io server initialized".
  A GET request to http://localhost:4006/api/chat/health returns status "UP".
result: [pending]

### 2. Socket Authentication Handshake
expected: |
  Connect a socket client (e.g., using the API tester) with a valid JWT token in the auth payload.
  The server should accept the connection and the client should receive a 'connect' event.
  Connecting without a token should result in an 'Authentication error'.
result: [pending]

### 3. Real-time Messaging
expected: |
  Join a room (conversationId) with two different socket clients.
  Client A sends a message. Client B should receive the 'new_message' event immediately.
  The message should also be persisted in the database.
result: [pending]

### 4. Typing Indicators
expected: |
  Client A emits a 'typing' event. Client B should receive a 'user_typing' event and display the TypingIndicator.
  Client A stops typing (or timeout occurs). Client B should receive 'user_stopped_typing' and the indicator should disappear.
result: [pending]

### 5. Vendor Dashboard Messages
expected: |
  Navigate to /vendor/chat in the web app.
  The conversation list should load. Selecting a conversation should load the message history in the right panel.
  Sending a message from the vendor dashboard should update the message list in real-time.
result: [pending]
