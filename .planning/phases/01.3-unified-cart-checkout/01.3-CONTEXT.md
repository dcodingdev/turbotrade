# Phase 1.3: Unified Cart & Checkout Flow - Context

**Gathered:** 2026-05-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Establishing the unified multi-vendor cart and checkout flow. This includes implementing persistent cart state (Zustand/localStorage), building a multi-step checkout UI, and integrating Stripe elements while handling real-time stock locking logic with graceful 409 Conflict resolutions.
</domain>

<decisions>
## Implementation Decisions

### Multi-vendor Cart Grouping
- **D-01:** Items in the cart must be grouped by vendor to make shipping and vendor-specific details clearer, rather than presenting a single flat list.

### Checkout Flow UI
- **D-02:** Implement a multi-page wizard flow (e.g., Cart -> Shipping -> Payment -> Review) rather than a single-page accordion, ensuring a clear and structured process for complex multi-vendor orders.

### Out-of-Stock / 409 Handling
- **D-03:** When real-time stock locking fails (409 Conflict), display an inline resolution modal first to inform the user, and then redirect them back to the cart so they can adjust their items.

### Stripe Elements Integration
- **D-04:** Use the unified Stripe Payment Element to handle multiple payment methods seamlessly instead of building custom card input fields.

</decisions>
