<task id="5" title="Implement Payment Step & Stock Conflict Handling">
<read_first>
- apps/web/src/components/modules/checkout/CheckoutWizard.tsx
- e:\DEV\Vs Code Program\ecommerce\services\order-service\src\modules\order.controller.ts
- e:\DEV\Vs Code Program\ecommerce\services\payment-service\src\modules\payments\payment.controller.ts
</read_first>
<action>
1. Create `apps/web/src/components/modules/checkout/PaymentStep.tsx`.
2. When the step mounts, call `POST /api/v1/orders` (via `fetch` or axios) using the cart items.
3. **Handle 409 Conflict Logic:** If the API returns a 400 error containing "Stock reservation failed", catch it. Display a Shadcn UI Dialog (Modal) explaining that stock is insufficient. Upon the user closing the modal, automatically redirect them back to the Cart Review step (step 0).
4. If order creation is successful, take the returned `orderId` and call `POST /api/v1/payments` with `{ orderId, amount, gateway: "STRIPE" }`.
5. Retrieve the `gatewayData.client_secret` from the response.
6. Initialize Stripe Elements with the `client_secret` and render `<PaymentElement />`.
7. Provide a "Pay Now" button that calls `stripe.confirmPayment()`. On success, clear the cart and show a success message.
</action>
<acceptance_criteria>
- Code contains explicit error catching for stock reservation failure that triggers a modal and redirects to step 0.
- `Elements` and `PaymentElement` from Stripe are used.
- Cart is cleared via `clearCart()` upon successful `stripe.confirmPayment()`.
</acceptance_criteria>
</task>
