# CONTEXT: Phase 3.2 — Advanced Catalog Management

## 1. Domain
Advanced vendor catalog management, focusing on binary media ingestion (ImageKit) and bulk data extraction (CSV).

## 2. Locked Requirements (from SPEC.md)
- [x] **R1-R3**: Drag-and-drop media uploads integrated with `product-service` (ImageKit).
- [x] **R4-R5**: Full and selection-based CSV export.
- [x] **Boundary**: Export-only for now; Bulk Import deferred.

## 3. Implementation Decisions

### 3.1 Media Upload Flow (Pre-upload)
- **Decision**: Images will be uploaded immediately upon being dropped/selected in the `ProductForm`.
- **Reasoning**: Provides immediate feedback to the user and simplifies the final "Save" request to a standard JSON body containing returned URLs.
- **Implementation**:
  - Add a temporary upload endpoint or use the existing `POST /api/v1/products` middleware logic adapted for a standalone `/uploads` route if needed.
  - Frontend component handles the "Uploading..." state and displays the thumbnail once the backend returns the ImageKit URL.

### 3.2 CSV Generation (Backend-driven)
- **Decision**: The CSV will be generated on the backend.
- **Reasoning**: Ensures consistency in data formatting and prepares the infrastructure for the future "Bulk Import" phase which will require heavy server-side processing.
- **Endpoint**: `GET /api/v1/products/export` (supports `ids[]` query param for selection).

### 3.3 Selection State Management (Reusable Hook)
- **Decision**: Implement row selection using a reusable `useDataTable` or `useSelection` hook pattern.
- **Reasoning**: The vendor dashboard will have multiple data-heavy tables (Orders, Billing, etc.). A reusable pattern ensures UI consistency and reduces boilerplate.
- **Pattern**: Hook manages `selectedIds` and provides helpers for "toggle all", "clear", etc.

### 3.4 CSV Formatting (Convention-based Flattening)
- **Decision**: Complex fields (Categories, Image Arrays) will be flattened using structured conventions.
- **Conventions**:
  - **Images**: Semicolon-delimited URLs in a single column (`mainImage;sub1;sub2`).
  - **Categories**: Comma-separated strings.
  - **Metadata**: Key-value pairs flattened into prefixed columns if necessary (e.g., `meta:color`, `meta:size`).
