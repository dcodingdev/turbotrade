# SPEC: Phase 3.2 — Advanced Catalog Management

## 1. Goal
Implement advanced catalog management tools for vendors, including a premium drag-and-drop media upload experience integrated with ImageKit and a comprehensive CSV export system for data backups and migrations.

## 3. Requirements

### 3.1 Media Uploads (Drag-and-Drop)
- **R1: Drag-and-Drop Area**: Replace/Supplement the existing Image URL input with a dedicated drop zone in `ProductForm.tsx`.
  - **Current State**: Manual URL text input only.
  - **Target State**: Interactive drop zone supporting `.jpg`, `.png`, and `.webp` with file preview.
  - **Acceptance**: Dropping a valid image file shows a thumbnail preview and clears the URL input (unless URL is manually re-entered).
- **R2: Hybrid Input Support**: Users can choose between uploading a file or providing a direct ImageKit/external URL.
  - **Current State**: URL only.
  - **Target State**: If a file is uploaded, the URL input is disabled or clearly secondary.
  - **Acceptance**: Form submission successfully handles either a `File` object or a `string` URL.
- **R3: Backend Integration**: Submit media as `multipart/form-data` to the existing `product-service` ImageKit middleware.
  - **Current State**: Form submits JSON with URL strings.
  - **Target State**: Form submits `FormData` if files are present.
  - **Acceptance**: Images are correctly uploaded to ImageKit and their URLs are persisted in MongoDB.

### 3.2 Bulk CSV Export
- **R4: Full Data Export**: Export function must generate a CSV containing all product fields (ID, Name, SKU, Description, Price, Stock, Category, Image URLs).
  - **Current State**: No export functionality.
  - **Target State**: A "Export CSV" button in the Inventory Header.
  - **Acceptance**: Clicking the button downloads a `.csv` file with headers matching the product schema.
- **R5: Selection-Based Export**: Allow exporting only specific products via checkbox selection in the inventory table.
  - **Current State**: Table has no selection mechanism.
  - **Target State**: Checkbox column in the DataTable; "Export Selected" appears in the action bar when rows are selected.
  - **Acceptance**: Selecting 3 rows and clicking export generates a CSV containing only those 3 rows.

## 4. Boundaries

### 4.1 In Scope
- UI components for Drag-and-Drop (e.g., using `react-dropzone` or native APIs).
- `FormData` submission logic in `useProductMutations`.
- CSV generation utility (client-side or server-side).
- Row selection logic for the Inventory DataTable.

### 4.2 Out of Scope
- **Bulk Import**: This phase focuses on *Export* only. Import will be a separate phase.
- **Image Editing**: No cropping or filters (ImageKit handles transformations via URL parameters if needed later).
- **Video Support**: Media is restricted to static images for this phase.

## 5. Acceptance Criteria
- [ ] `ProductForm` allows dragging an image which then appears as a preview.
- [ ] Product creation works with both direct file uploads and URL inputs.
- [ ] CSV export from the header downloads all products in the current view.
- [ ] CSV export from selection downloads exactly the selected products.
- [ ] Exported CSV can be opened in Excel/Google Sheets with correct column alignment.
