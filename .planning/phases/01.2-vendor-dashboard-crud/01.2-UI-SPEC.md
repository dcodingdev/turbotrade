# Phase 1.2: Vendor Dashboard UI Specification

## Design System Integration
Phase 1.2 will extend the core design system from `01.1-UI-SPEC.md` but with a focus on data density and operational efficiency.

### Color Palette (Operational Context)
- **Primary**: `var(--primary)` (OKLCH from globals.css) for CTAs.
- **Surface**: `var(--card)` for dashboard cards and table rows.
- **Border**: `var(--border)` (subtle contrast).
- **Destructive**: `var(--destructive)` for delete actions.

### Layout: Dashboard Shell
- **Sidebar**:
  - Width: `280px` (collapsed: `80px` for mobile).
  - Background: `var(--card)`.
  - Links: Icon (Lucide) + Label.
- **Header**:
  - Sticky at `top-0`.
  - Breadcrumbs navigation.
  - User profile / Logout dropdown.

### Components: Data Tables
- **Standard shadcn/ui Table**:
  - Zebra striping (optional, preferred subtle).
  - Cell padding: `px-4 py-3`.
  - Header font: Bold, small-caps or uppercase tracking.

### Components: Forms
- **Input Fields**:
  - Border radius: `var(--radius-lg)`.
  - Focus state: `ring-2 ring-primary`.
- **Validation**:
  - Inline error messages in `var(--destructive)`.
  - Button state: Disable and show spinner during submission.

### Spacing & Grid
- **Dashboard Container**: `p-8` (XL scale).
- **Card Gaps**: `gap-6` (LG scale).

### Typography
- **Dashboard Headers**: `text-2xl font-bold`.
- **Table Labels**: `text-xs font-semibold uppercase text-muted-foreground`.

## Visual Checkpoints
- [ ] Sidebar is responsive and hides/collapses on mobile.
- [ ] Product forms use clear labels and have accessible error states.
- [ ] CRUD actions are preceded by clear visual indicators (Toasts).
- [ ] Theme toggle correctly affects the data-dense table and form elements.
