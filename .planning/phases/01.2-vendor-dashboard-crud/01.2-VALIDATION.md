# Phase 1.2: Vendor Dashboard Validation Criteria

## Functional Verification

### 1. Dashboard Layout
- **Test**: Navigate to `/vendor`.
- **Expected**: Sidebar is visible, links work (Home, Inventory, Billing), and user can toggle theme.

### 2. Product Listing
- **Test**: Navigate to `/vendor/inventory`.
- **Expected**: Table loads products specific to the current vendor. Shows "No products" if empty.

### 3. Create Product
- **Test**: Click "Add Product", fill form correctly, and submit.
- **Expected**: Success toast appears, modal closes, and new product appears in the table.

### 4. Form Validation
- **Test**: Try to submit an empty product form.
- **Expected**: Field-level validation errors appear (e.g., "Price must be positive").

### 5. Update Product
- **Test**: Edit an existing product's price.
- **Expected**: Changes reflect in the table immediately after success toast.

### 6. Delete Product
- **Test**: Click Delete on a product row and confirm.
- **Expected**: Product is removed from the table; success toast confirms deletion.

## Technical Verification
- [ ] No memory leaks on table re-renders.
- [ ] Zod schema handles all edge cases (negative prices, empty descriptions).
- [ ] TanStack Query cache is invalidated correctly after mutations.
